Text Pop

⇦ ⇨

  Text Pop was a game I made using Unity and C#. The goal of the game is to avoid obstacles by moving your mouse to control the player and pressing keys to pop incoming hazards. (The guide to the gameplay is in one of the images above). The game features multiple difficulty levels and features a high score system for all three difficulties. The longer you survive the more points you get!   My goal for Text Pop was to create a game that I could solely make myself. I wanted to put my Unity knowledge and programming skills to the test and see if I could make a simple game that was challenging and fun to play. Due to my knowledge of C# I was able to program a significant amount of the project relatively quickly but near the end of the development I ran into the ninety-ninety rule. Essentially, when I thought I was nearly done with the everything, I learned the hard way that finalizing the game by building and running the application takes a significant amount of time in itself and, like the ninety-ninety rule says, took just as much time finalizing the game as it did building the entire thing.   Despite the challenges of the project, I am very happy with the final product. The game is very simplistic but it was the first game I’ve made that has features you’d see in other games like music, settings, high scores, and etc. Nevertheless, my favorite part of the project, by far, is the look. The color of each hazard is associated with an RGB value that changes based on the duration of the game, the speed of the hazard, and the value of its key. That means each hazard has a unique color that even changes as the game progresses. I thought this made the game look visually appealing and even helps the player keep track of how long they have been playing. While Text Pop is a finished game, if I were to add anything, I would add an achievement system where the player can unlock different colors based on their high scores for each difficulty mode.
The Github repository of the game is here and you can play it yourself by downloading it here (Windows only)!

First Person Prototype


  Some of my fondest memories come from playing around with Discord bots that could do all kinds of really cool commands. However, as time passed, almost all the bots I used to use were discontinued or out of date. That was the reason why I decided to make Jack, a Discord bot that could do a plethora of different commands from shapeshifting into certain users to parroting dialogue.   Jack was made using Javascript, Node.js, and discord.js. The hardest part of the project was just getting used to Node.js and Javascript as, at the time, I had very little experience with both. With some time, I was able to get used to everything and eventually started implementing discord.js to get the bot running and functional.   Jack was made with scaling in mind so it could feature a large number of different commands without slowing down too much or negatively affecting other commands. In fact, Jack was also made so adding a new command would be isolated from the bot’s main operations so there would not be any weird or unexpected effects from giving the bot a new feature. My favorite part of the bot is definitely its ability to make memes instantly. All you would need to do is pick the template, choose what you want the meme to say, and Jack will output it in Discord. I added all the features I could think of already but in the future, I look forward to thinking of new commands and adding it to the bot whenever I can!
The Github repository of Jack is here!

Clubs

Here, I'll go over the relevant clubs I attend at Rutgers University! I will give a short description of the club then go over a project I worked on in each club.

COGS

COGS, or the Creation of Games Society, is a club focused on creating a community where artists, musicians, game designers, and programmers can come together to create video games. Members usually work on one big project throughout the semester while sometimes there are events such as Game Jams or Firesides where people can be challenged to create games with certain themes or rules. For example, for the 2021 Scarlet Knights Game Jam, we had one week to create a game surrounding the theme of jam and had to create the project using Unity and C#. To follow the theme, my team decided that it would be fun to create a platformer where we have two players with different abilities. One player could jam out with a guitar and send sound waves while the other player could throw jam to create terrain that could bounce players, enemies, and sound waves. The focus of the game was to create fun and unique puzzles that could only be solved by having both players use their abilities cooperatively. This event was very challenging but also a ton of fun and I learned a significant amount about working with different people with different skillsets. If you want to learn more about Jammin' Jelly, the Github repository of the game is here!

Blueprint

Blueprint seeks to promote innovation and creativity by bringing together groups of product managers, designers, and engineers to build products. Groups are student led and focus on creating one application throughout a semester/school year. For example, a project I worked on with another engineer was a cooking quiz app. The quiz was made in Java using Android Studio and was essentially a small app that you would answer cooking questions with. It featured questions like What is the world's most expensive spice? (as of 2021 - saffron) and would keep track how many questions the user answered correctly. This app was actually the first time I worked simultaneously with another programmer using Github and I learned how to more effectively program with a partner. If you want to learn more about the cooking quiz, the Github repository of the app is here!
