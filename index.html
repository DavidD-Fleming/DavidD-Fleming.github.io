Pokemon Decoy (High School)


  Pokemon decoy was a project I worked on in high school with Benjamin Lifshey. The game used Replit as the platform of the website and Javascript (with HTML and CSS as the basis) as the primary language. As the name suggests, the game was made to mimic Pokemon with two players going against each other with the goal of setting the enemy's health bar to 0. Player 1 (Triangle) has the four abilities seen above while the Attempted Pentagon has two abilities being an ice beam and a heal of its own.   This was the first project I programmed that I was really happy about. It was the first time I programmed in Javascript and I learned a significant amount about the basis of coding. While I do not remember struggling with anything in particular, looking back at the program and the actual game - the whole thing was terribly inefficient. Looking past the fact that there is no indication about how to use each player's abilities and the website was not built with different screen sizes in mind, the organization and execution of the code left many areas prone to bugs and could have been faster if any methods were used.   Looking past the inefficiencies, my favorite part of the project was the sheer attention to detail. Buttons light up when pressed and each player's health bar would shake when they take damage. Also when players landed a critical strike it would be displayed in Japanese. Also the game could be played in Japanese. If I were to come back to the project, I would definitely build it in another language that fit game development better like C#.
I believe the website is still running and, if it is, you can access it here. Use QWER (in that order) for Triangle’s abilities and AS for Attempted Pentagon’s abilities.

Jammin' Jelly (Scarlet Game Jam)


  Jammin’ Jelly was a game I made in a week for the 2021 Scarlet Knights Game Jam. The game was made in Unity using C# and I worked together with a game designer and animator. The theme of the Game Jam was “jam” so we thought it would be fun to create a game where we have two players and one player could jam out with a guitar and send sound waves while the other player could throw jam to create terrain that can bounce players, enemies, and sound waves. Both players have to use their abilities and work together to pass enemies and progress until they reach the end.   This project taught me a lot about C# and Unity but it primarily helped me learn what it was like to work with a team as a programmer. There were definitely communication issues that delayed progress and made it much harder to code. We also found our animator really late into the Game Jam so we were not able to implement the visuals of the game in time. This meant that while the game was actually really fun and had really interesting puzzle solving opportunities - it looked boring. The primary thing I learned from this project was to always, always communicate and that nitty gritty details are vital when it comes to software that has delicate interactions with each other and the player.   Despite the issues of the project, I am really happy about the mechanics of the game. The players had solid movement and all the abilities (sound waves, jam terrain) worked perfectly together. I was also really happy with the enemies as they were all dynamically made so moving enemies, enemies that shot projectiles just needed a small numbers adjustment if I wanted to duplicate them and move them around. If we had more time in the Game Jam or if I were to come back to this game, I would implement visuals and add more levels to the game.
The Github repository of the game is here.

Cooking Quiz


  For my Blueprint club, I worked with another programmer to create a cooking quiz app. The project was made in Java using Android Studio and was essentially a small app that you would answer cooking questions with. It was the first time I worked collaboratively with another programmer and the first time I used Android Studio. Nevertheless, while there were a few complications and errors here and there, it was overall a very solid experience where everything went pretty smoothly. We had a few issues with Github since it was both our first times sharing a repository, but we were able to solve them relatively quickly. My favorite part of the project was figuring out how to use Android Studio and seeing the app work on an emulator felt really rewarding. If I were to come back to this project, I would add a significant amount of new questions and maybe even look to expand and add features such as comparing results with friends, adding more quiz topics, and etc.
The Github repository of the app is here.
