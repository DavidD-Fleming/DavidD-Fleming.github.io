Text Pop

⇦ ⇨

  Text Pop was a game I made using Unity and C#. The goal of the game is to avoid obstacles by moving your mouse to control the player and pressing keys to pop incoming hazards. (The guide to the gameplay is in one of the images above). The game features multiple difficulty levels and features a high score system for all three difficulties. The longer you survive the more points you get!   My goal for Text Pop was to create a game that I could solely make myself. I wanted to put my Unity knowledge and programming skills to the test and see if I could make a simple game that was challenging and fun to play. Due to my knowledge of C# I was able to program a significant amount of the project relatively quickly but near the end of the development I ran into the ninety-ninety rule. Essentially, when I thought I was nearly done with the everything, I learned the hard way that finalizing the game by building and running the application takes a significant amount of time in itself and, like the ninety-ninety rule says, took just as much time finalizing the game as it did building the entire thing.   Despite the challenges of the project, I am very happy with the final product. The game is very simplistic but it was the first game I've made that has features you'd see in other games like music, settings, high scores, etc. Nevertheless, my favorite part of the project, by far, is the look. The color of each hazard is associated with an RGB value that changes based on the duration of the game, the speed of the hazard, and the value of its key. That means each hazard has a unique color that even changes as the game progresses. I thought this made the game look visually appealing and even helps the player keep track of how long they have been playing. While Text Pop is a finished game, if I were to add anything, I would add an achievement system where the player can unlock different colors based on their high scores for each difficulty mode.
The Github repository of the game is here and you can play it yourself by downloading it here (Windows only).

First Person Prototype


  This first person prototype was built using Unity and C#. The goal of the project is to fix the code used by Brackeys (popular Unity youtuber) for his first person movement video and implement abilities used by Jett (from Valorant) to test its functionality. The foundation of the code mirrors the one used by Brackeys but almost all of it was reworked so movement wouldn't be as clunky and remove bugs such as the player being able to walk into the air. Then, there is an additional script that allows the player to access abilities used by Jett such as dashing, throwing knives, throwing smokes, etc.   The most challenging part of the project is also my favorite, which is implementing Jett's abilities. While most of them are pretty straightforward, she has one where she throws a smoke ball that expands when it collides with terrain. The difficult part of the smoke ball is that the player is able to guide it while it is flying in the air. Long story short, I had to learn and use quaternions which is a high level physics mechanic in order to get the smoke ball to rotate and move appropriately. If I were to come back to this project, I would utilize this prototype as the foundation to create movement for a fully realized first person game.
The Github repository of this first person prototype is here.

Discord Bot


  What started as a fun little project to create a Discord bot that I could use with friends turned into an endeavor that significantly enhanced my knowledge of back end development. My Discord bot, or Jack Bot, was made using Javascript, Node.js, and discord.js. Eventually, as I wanted to give my bot more ambitious features, I also ended up utilizing RESTful apis and JSON. The bot features a wide variety of commands such as dming other users, creating customizable memes, and changing into other users.   Jack Bot was made with scaling in mind so it could feature a large number of different commands without slowing down too much or negatively affecting other commands. In fact, Jack Bot was also made so adding a new command would be isolated from the bot's main operations so there would not be any weird or unexpected effects from giving the bot a new feature.   One of the challenges I ran into while making this Discord bot was creating more advanced commands. I wanted to go further than just making the bot do things like simple arithmetic or outputting random numbers. In order to be more adventurous I had to teach myself a significant amount about promises, apis, JSON, and etc. While it was challenging connecting the pieces, I was eventually able to understand enough that Jack Bot is now able to do things like defining words using Urban Dictionary and sending inspirational quotes from Kanye West. While I am happy with the amount of commands Jack Bot has, I look forward to consistently adding new features to make it even more fun.
The Github repository of Jack Bot is here.

COGS

COGS, or the Creation of Games Society, is a club focused on creating a community where artists, musicians, game designers, and programmers can come together to create video games. Members usually work on one big project throughout the semester while sometimes there are events such as Game Jams or Firesides where people can be challenged to create games with certain themes or rules. For example, for the 2021 Scarlet Knights Game Jam, we had one week to create a game surrounding the theme of jam and had to create the project using Unity and C#. To follow the theme, my team decided that it would be fun to create a platformer where we have two players with different abilities. One player could jam out with a guitar and send sound waves while the other player could throw jam to create terrain that could bounce players, enemies, and sound waves. The focus of the game was to create fun and unique puzzles that could only be solved by having both players use their abilities cooperatively. This event was very challenging but also a ton of fun and I learned a significant amount about working with different people with different skillsets. If you want to learn more about Jammin' Jelly, the Github repository of the game is here.

Blueprint

Blueprint seeks to promote innovation and creativity by bringing together groups of product managers, designers, and engineers to build products. Groups are student led and focus on creating one application throughout a semester/school year. For example, a project I worked on with another engineer was a cooking quiz app. The quiz was made in Java using Android Studio and was essentially a small app that you would answer cooking questions with. It featured questions like What is the world's most expensive spice? (as of 2021 - saffron) and would keep track how many questions the user answered correctly. This app was actually the first time I worked simultaneously with another programmer using Github and I learned how to more effectively program with a partner. If you want to learn more about the cooking quiz, the Github repository of the app is here.
